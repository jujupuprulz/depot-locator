Security Policy

We take the security of the Depot Locator application seriously. This document outlines our security practices and how to report security vulnerabilities.

Data Security

The Depot Locator application stores data in the following ways:

  • Local browser storage for temporary data
  • Supabase database for persistent storage

All data is transmitted securely using HTTPS. We do not collect or store personal information beyond what is necessary for the application to function.

Security Measures

We implement the following security measures:

  • Content Security Policy (CSP) to prevent XSS attacks
  • HTTP security headers to protect against common web vulnerabilities
  • HTTPS-only access to ensure encrypted data transmission
  • Regular security updates and dependency maintenance

Reporting Security Issues

If you discover a security vulnerability, please report it by emailing jswtsao@gmail.com. We request that you:

  • Provide detailed information about the vulnerability
  • Allow us reasonable time to address the issue before public disclosure
  • Avoid accessing or modifying user data during your investigation

Updates to This Policy

This security policy may be updated from time to time. The latest version will always be available on this page.
